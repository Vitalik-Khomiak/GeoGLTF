# GeoGLTF Viewer

Цей файл є короткою живою документацією проєкту. Його задача не перераховувати кожну дрібну утиліту в коді, а стисло і зрозуміло фіксувати:

- що вміє програма
- з яких основних частин вона складається
- які ключові функції за що відповідають
- які важливі обмеження є в поточній реалізації

## 1. Що це за програма

`GeoGLTF Viewer` — це навчальний вебзастосунок для перегляду 3D-моделей у форматі `.glb`.

Технології:

- `HTML`
- `CSS`
- `JavaScript`
- `Three.js`

Основна ідея програми — дати учню або викладачу простий інструмент для візуального аналізу просторових фігур у браузері.

## 2. Що вміє програма

На поточний момент програма підтримує:

- перегляд `.glb` моделей у 3D-сцені
- локальне завантаження моделей через кнопку
- drag-and-drop завантаження файлів
- бібліотеку опублікованих моделей з `assets/library.json`
- тимчасову локальну бібліотеку моделей поточної сесії
- автоматичне прев'ю моделей у списку
- перемикання між бібліотекою і режимом перегляду
- скидання камери і автоцентрування моделі
- адаптивний 3D-viewer для телефона з повторним центруванням моделі після resize
- координатну сітку
- осі `X / Y / Z` як кольорові напрямні лінії в сцені
- mini-gizmo орієнтації з 3D-стрілками напрямків осей
- звичайний режим показу моделі
- математичний режим показу
- wireframe-режим
- підрахунок базової геометричної статистики
- режим `Розгортка` для окремих навчальних фігур
- базовий захист від порожніх або некоректних `.glb`

## 3. Поточні моделі в бібліотеці

Зараз у проєкті є такі опубліковані моделі:

- `Куб`
- `Куб зі зрізом`
- `Циліндр`
- `Піраміда`
- `Сфера`

## 4. Структура проєкту

Основні файли:

- `index.html` — структура інтерфейсу
- `styles.css` — стилі застосунку
- `src/app.js` — основна логіка програми
- `assets/library.json` — каталог моделей
- `assets/models/*.glb` — 3D-моделі
- `README.md` — короткий опис проєкту
- `PROGRAM_FUNCTIONS.md` — цей файл

## 5. Як працює програма

Після запуску сторінки програма:

1. створює `Three.js` сцену
2. ініціалізує рендерер, камеру та керування
3. додає світло, сітку, осі й mini-gizmo
4. підключає всі кнопки, перемикачі та drag-and-drop події
5. читає `assets/library.json`
6. показує бібліотеку моделей
7. запускає постійний цикл рендеру

Коли користувач відкриває модель:

1. файл зчитується в пам'ять
2. модель парситься через `GLTFLoader`
3. попередня модель очищається зі сцени
4. нова модель додається в сцену
5. програма центрує її
6. камера підлаштовується під розмір моделі
7. оновлюються статистика, статус і доступні режими

## 6. Ключові блоки логіки

Уся основна логіка зараз зосереджена в `src/app.js`.

Головні логічні блоки:

- ініціалізація сцени
- бібліотека моделей
- завантаження та парсинг `.glb`
- автоцентрування і кадрування камери
- режими відображення
- режим розгортки
- генерація прев'ю
- очищення ресурсів
- адаптація інтерфейсу

## 7. Основні функції програми

Нижче не подано кожну дрібну допоміжну функцію окремо. Тут зібрано саме ключові функції, які формують основну поведінку програми.

### 7.1. Ініціалізація сцени

- `createRenderer(...)`
  Створює `WebGLRenderer`.

- `createCamera()`
  Створює основну камеру.

- `createControls(...)`
  Налаштовує `OrbitControls`.

- `initializeScene()`
  Готує сцену, фон, освітлення і базову 3D-обстановку.

### 7.2. Навігація та gizmo

- `initializeViewGizmo()`
  Створює mini-gizmo орієнтації.

- `createGizmoAxis(...)`
  Малює одну вісь gizmo.

- `renderViewGizmo()`
  Відображає gizmo поверх основної сцени.

### 7.3. Бібліотека моделей

- `loadPublishedLibrary()`
  Читає `assets/library.json` і формує список опублікованих моделей.

- `normalizePublishedAsset(...)`
  Приводить запис із JSON до внутрішнього формату.

- `renderAssetLibraries()`
  Перемальовує списки моделей у UI.

- `renderAssetList(...)`
  Створює картки моделей у бібліотеці.

- `registerSessionFiles(...)`
  Додає локальні `.glb` файли в сесійну бібліотеку.

### 7.4. Завантаження моделей

- `loadAsset(...)`
  Основна функція відкриття моделі.

- `resolveAssetArrayBuffer(...)`
  Вирішує, звідки читати дані моделі: з локального файлу чи з каталогу сайту.

- `parseModelBuffer(...)`
  Парсить `.glb`, додає модель у сцену та оновлює UI.

- `containsRenderableMesh(...)`
  Перевіряє, чи містить модель реальну геометрію для показу.

### 7.5. Підготовка сцени під модель

- `prepareModel(...)`
  Готує mesh-об'єкти до показу.

- `normalizeModelTransform(...)`
  Центрує модель у сцені.

- `frameCurrentModel(...)`
  Підлаштовує камеру під модель.

- `updateGridScale(...)`
  Масштабує сітку та осі під розмір фігури.

### 7.6. Режими відображення

- `applyMathStyleMode(...)`
  Вмикає або вимикає математичний стиль подачі.

- `applyWireframeMode(...)`
  Вмикає або вимикає wireframe.

- `syncRenderModeControls()`
  Узгоджує режими між собою, щоб вони не конфліктували.

### 7.7. Розгортка

- `setUnfoldModeEnabled(...)`
  Вмикає або вимикає режим розгортки.

- `toggleUnfoldPlayback()`
  Запускає або зупиняє анімацію розгортання.

- `updateUnfoldAnimation(...)`
  Оновлює стан анімації в кожному кадрі.

- `refreshUnfoldController()`
  Перебудовує розгортку після зміни моделі.

- `buildCubeUnfoldController(...)`
  Формує розгортку куба.

- `buildSquarePyramidUnfoldController(...)`
  Формує розгортку піраміди.

### 7.8. Прев'ю моделей

- `enqueueThumbnail(...)`
  Додає модель у чергу на побудову прев'ю.

- `processThumbnailQueue()`
  Генерує прев'ю послідовно.

- `createAssetThumbnail(...)`
  Створює thumbnail моделі.

### 7.9. Статус і статистика

- `collectModelStats(...)`
  Рахує вершини і трикутники.

- `updateStats(...)`
  Оновлює блок інформації про модель.

- `setStatus(...)`
  Показує поточний стан програми.

### 7.10. Очищення та стабільність

- `disposeActiveModel()`
  Очищає активну модель зі сцени.

- `disposeUnfoldController()`
  Очищає допоміжну сцену розгортки.

- `disposeThreeObject(...)`
  Очищає тимчасові `Three.js` об'єкти.

- `disposeMaterial(...)`
  Звільняє матеріали й текстури.

### 7.11. Адаптивність і цикл рендеру

- `resizeRenderer()`
  Підлаштовує renderer під розмір контейнера.

- `syncViewerLayout(...)`
  Перераховує layout viewer після змін інтерфейсу та повторно центрує модель, коли мобільний viewport змінює реальний розмір.

- `syncMobileViewportHeight()`
  Коригує висоту viewer на мобільних пристроях під реальну видиму частину екрана.

- `animate()`
  Основний цикл рендеру сцени.

## 8. Поточний стан розгорток

Підтримується:

- `Куб`
- `Піраміда`

Поки не підтримується:

## 9. Важливі обмеження

### 9.1. Не кожну 3D-модель можна коректно розгорнути

Для навчальних многогранників розгортка працює добре, бо фігура складається з плоских граней.

Для довільних моделей ситуація значно складніша:

- треба виділити окремі грані
- знайти спільні ребра між ними
- побудувати дерево розрізів
- уникнути накладання граней на площині
- зберегти зрозумілу навчальну форму розгортки

### 9.2. Сфера не має простої точної розгортки

Сферу не можна точно розгорнути на площину без деформації.

Тобто:

- можна зробити лише наближення
- або ділити сферу на сегменти
- або показувати "пелюстки"
- або використовувати картографічні проєкції

Тому для сфери можна зробити:

- навчальну умовну розгортку
- сегментовану розгортку
- анімований перехід до набору меридіональних смуг

Але це вже не буде "чистою" геометрично точною розгорткою, як для куба чи піраміди.

### 9.3. Розгортка всіх моделей — складна окрема задача

Щоб підтримати розгортку для всіх моделей, потрібен окремий геометричний модуль, який буде:

- аналізувати топологію меша
- знаходити допустимі лінії розрізу
- будувати площинне розміщення граней
- контролювати самоперетини
- враховувати, що частина моделей взагалі не має точної розгортки

Практично це означає:

- для простих многогранників задача реальна
- для довільних `.glb` задача складна
- для криволінійних тіл, як-от сфера, задача або наближена, або математично неточна

## 11. Висновок

Програма вже вміє:

- показувати навчальні 3D-моделі
- працювати з локальними і вбудованими `.glb`
- перемикати режими подачі
- будувати базові розгортки для окремих фігур
- безпечніше обробляти проблемні моделі

Найбільш реалістичний шлях розвитку:

- додавати розгортки для окремих навчальних тіл
- не намагатися одразу автоматично розгортати всі можливі `.glb`
- для сфери робити окрему спеціальну навчальну візуалізацію, а не "справжню" класичну розгортку
