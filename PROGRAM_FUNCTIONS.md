# Опис програми GeoGLTF Viewer

Цей файл є живою документацією проєкту. Надалі всі зміни в логіці програми варто відображати тут, щоб список функцій і можливостей залишався актуальним.

## 1. Загальний опис програми

`GeoGLTF Viewer` — це статичний вебзастосунок для перегляду 3D-моделей у форматі `.glb` у браузері. Програма побудована на:

- `HTML`
- `CSS`
- `JavaScript`
- `Three.js`

Основне призначення програми — навчальний перегляд геометричних 3D-моделей для аналізу просторових фігур, осей координат, сітки, форми об'єктів і режимів подачі моделі.

## 2. Структура проєкту

### Основні файли

- `index.html` — структура інтерфейсу
- `styles.css` — стилі застосунку
- `src/app.js` — основна логіка програми
- `assets/library.json` — каталог опублікованих моделей
- `assets/models/*.glb` — 3D-моделі
- `README.md` — короткий опис проєкту та запуску
- `PROGRAM_FUNCTIONS.md` — цей файл із детальним описом програми та всіх функцій

## 3. Що є в програмі

Програма вже містить такі можливості:

- перегляд `.glb` моделей у 3D-сцені
- завантаження локальних `.glb` файлів із комп'ютера
- drag-and-drop завантаження моделі в сцену
- бібліотеку опублікованих моделей з `assets/library.json`
- локальну тимчасову бібліотеку моделей поточної сесії
- автоматичне створення прев'ю для карток моделей
- перемикання між режимом бібліотеки та режимом перегляду
- скидання камери та автоматичне центрування моделі
- координатну сітку
- осі `X/Y/Z`
- навігаційний mini-gizmo орієнтації
- звичайний режим відображення
- математичний стиль відображення
- wireframe-режим
- підрахунок вершин і полігонів
- показ статусу завантаження
- показ назви моделі та її розміру
- збереження стану прихованої підказки через `localStorage`
- адаптацію інтерфейсу під мобільний перегляд

## 4. Як працює програма

Після відкриття сторінки застосунок:

1. створює `Three.js` сцену
2. створює рендерер, камеру та керування
3. додає світло, сітку, осі та mini-gizmo
4. підключає події кнопок, drag-and-drop і перемикачів
5. рендерить порожні бібліотеки моделей
6. читає `assets/library.json`
7. формує картки моделей
8. запускає нескінченний цикл рендеру сцени

Коли користувач обирає модель:

1. програма визначає джерело моделі
2. зчитує `.glb` файл в `ArrayBuffer`
3. парсить його через `GLTFLoader`
4. очищає попередню модель зі сцени
5. додає нову модель
6. центрує її відносно координатної сцени
7. підлаштовує камеру
8. оновлює статистику
9. за потреби застосовує математичний стиль або wireframe

## 5. Глобальні частини програми

У `src/app.js` логіка побудована навколо таких основних сутностей:

- DOM-елементи інтерфейсу
- `scene` — головна 3D-сцена
- `renderer` — WebGL-рендерер
- `camera` — перспективна камера
- `controls` — `OrbitControls`
- `loader` — завантажувач GLB-моделей
- `thumbnailLoader` — окремий завантажувач для прев'ю
- `gridHelper` — координатна сітка
- `axesHelper` — осі координат
- `gizmoScene`, `gizmoCamera`, `gizmoRoot` — mini-gizmo
- `publishedAssets` — список моделей із JSON
- `sessionAssets` — список локально доданих моделей
- `activeModelRoot` — поточна модель на сцені
- `activeAssetId` — ідентифікатор активної моделі
- `savedCameraState` — збережений ракурс камери

## 6. Повний список функцій

Нижче наведено актуальний список усіх функцій із `src/app.js` та їх призначення.

### 6.1. Ініціалізація сцени

#### `createRenderer(targetCanvas)`

Створює `THREE.WebGLRenderer` для основного полотна сцени.

Що робить:

- прив'язує renderer до `canvas`
- вмикає згладжування
- вмикає прозорість фону
- підлаштовує `pixelRatio`
- встановлює `SRGBColorSpace`
- вимикає автоматичне очищення кадру, бо окремо домальовується gizmo

#### `createCamera()`

Створює перспективну камеру.

Що робить:

- створює `PerspectiveCamera`
- задає стартову позицію камери

#### `createControls(targetCamera, domElement)`

Налаштовує `OrbitControls`.

Що робить:

- вмикає плавність руху
- задає швидкості обертання, панорамування та зуму
- задає поведінку кнопок миші
- встановлює стартову ціль камери

#### `initializeScene()`

Виконує базове налаштування сцени.

Що робить:

- задає фон сцени
- додає м'яке загальне освітлення
- додає ключове направлене світло
- додає контрове світло
- ініціалізує mini-gizmo
- запускає первинне масштабування renderer

### 6.2. Mini-gizmo

#### `initializeViewGizmo()`

Створює окрему маленьку сцену для навігаційного gizmo.

Що робить:

- задає камеру gizmo
- додає світло в gizmo-сцену
- створює осі `X`, `Y`, `Z`

#### `createGizmoAxis(label, color, direction)`

Створює одну вісь gizmo.

Що містить:

- лінію осі
- кольорову сферу на кінці
- текстову мітку осі

#### `createAxisLabelTexture(text, color)`

Створює текстуру для літери осі через `canvas`.

Що робить:

- малює круглу білу підкладку
- малює кольорову обводку
- малює літеру осі в центрі
- повертає `CanvasTexture`

### 6.3. Інформація про оновлення коду

#### `updateProjectUpdatedLabel()`

Оновлює текст із датою останнього оновлення коду.

Що робить:

- пробує визначити GitHub-репозиторій
- звертається до GitHub API по останній коміт
- бере дату коміту
- якщо API недоступне, використовує `document.lastModified`

#### `resolveGitHubRepoInfo()`

Визначає `owner` і `repo`, якщо застосунок запущений через GitHub Pages.

#### `formatDisplayDate(dateInput)`

Форматує дату в українському форматі.

#### `getLocalFallbackDate()`

Повертає резервну локальну дату оновлення.

### 6.4. Події інтерфейсу

#### `bindEvents()`

Підписує всі елементи інтерфейсу на події.

Що підключає:

- `resize`
- `orientationchange`
- `visualViewport.resize`
- кнопки скидання виду
- кнопку закриття підказки
- кнопку повернення до бібліотеки
- кнопку наступної моделі
- перемикачі `math`, `grid`, `axes`, `wireframe`
- drag-and-drop події
- клавішу `R`
- відстеження руху курсора по canvas

#### `onFileInputChange(event)`

Обробляє вибір файлів через поле `input`.

#### `onDragEnter(event)`

Підсвічує зону drop при вході курсора з файлом.

#### `onDragOver(event)`

Дозволяє drop і підтримує активний стиль drop-зони.

#### `onDragLeave(event)`

Прибирає підсвічування drop-зони.

#### `onDrop(event)`

Приймає перетягнуті файли та передає їх на реєстрацію.

### 6.5. Бібліотека моделей

#### `registerSessionFiles(files)`

Реєструє локальні моделі в межах поточної сесії.

Що робить:

- залишає тільки `.glb`
- відкидає невалідні файли
- перевіряє дублікати
- додає нові моделі в `sessionAssets`
- оновлює список у UI

#### `loadPublishedLibrary()`

Завантажує каталог моделей з `assets/library.json`.

Що робить:

- читає JSON
- бере масив `assets`
- нормалізує кожен запис
- зберігає в `publishedAssets`
- оновлює інтерфейс

#### `normalizePublishedAsset(entry, index)`

Приводить запис із JSON до внутрішнього формату asset.

#### `renderAssetLibraries()`

Перемальовує два списки:

- опубліковані моделі
- локальні моделі сесії

#### `renderAssetList(container, assets, emptyMessage)`

Створює DOM-картки для моделей.

Що робить:

- очищає контейнер
- показує пустий стан, якщо список порожній
- створює кнопку-картку для кожної моделі
- показує прев'ю або placeholder
- ставить задачу генерації thumbnail
- вішає клік на завантаження моделі

### 6.6. Завантаження та перемикання моделей

#### `loadAsset(asset, options = {})`

Головна функція завантаження моделі.

Що робить:

- робить модель активною
- очищає попередню модель
- оновлює заголовок viewer
- за потреби перемикає застосунок у режим перегляду
- показує статус завантаження
- завантажує буфер моделі
- парсить модель
- оновлює layout
- обробляє помилки

#### `getAllAssets()`

Повертає спільний список усіх моделей.

#### `loadNextAsset()`

Завантажує наступну модель по колу.

#### `switchToLibraryMode()`

Перемикає інтерфейс у режим бібліотеки.

#### `switchToViewerMode()`

Перемикає інтерфейс у режим перегляду моделі.

#### `updateViewerHeader(title)`

Оновлює заголовок активної моделі у viewer.

#### `updateViewerActions()`

Оновлює доступність кнопок навігації, зокрема кнопки переходу до наступної моделі.

### 6.7. Завантаження буфера моделі

#### `readFileAsArrayBuffer(file)`

Читає локальний файл у форматі `ArrayBuffer`.

#### `fetchAssetArrayBuffer(filePath)`

Завантажує `.glb` файл із каталогу проєкту через `fetch`.

#### `resolveAssetArrayBuffer(asset)`

Уніфікує джерело даних:

- якщо модель опублікована, використовує `fetch`
- якщо локальна, використовує `FileReader`

### 6.8. Парсинг і підготовка моделі

#### `parseModelBuffer(arrayBuffer, asset, options = {})`

Парсить GLB-модель через `GLTFLoader`.

Що робить:

- створює `gltf.scene`
- робить його активною моделлю
- готує модель до відображення
- центрує модель
- додає модель у сцену
- застосовує режими відображення
- підлаштовує камеру
- рахує статистику
- оновлює UI

#### `prepareModel(modelRoot)`

Підготовляє mesh-об'єкти моделі.

Що робить:

- проходить по всіх вузлах
- для mesh-об'єктів вмикає тіні
- зберігає оригінальні матеріали
- переводить матеріали в `DoubleSide`

#### `normalizeModelTransform(modelRoot)`

Нормалізує положення моделі в сцені.

Що робить:

- обчислює межі моделі
- знаходить центр
- зміщує модель так, щоб:
  - вона стояла на площині `Y = 0`
  - була центрована по `X/Z`

### 6.9. Камера та кадрування

#### `updateSavedCameraState()`

Запам'ятовує поточний ракурс камери відносно моделі.

#### `applySavedCameraState(modelCenter, modelSize, minimumDistance = 0)`

Відновлює збережений ракурс для нової або поточної моделі.

#### `getFitCameraDistance(bounds, viewDirection)`

Обчислює відстань камери, достатню для повного вміщення моделі в кадр.

#### `frameCurrentModel(options = {})`

Центральна функція автоматичного підбору виду.

Що робить:

- якщо моделі немає, повертає стартовий ракурс
- якщо модель є:
  - обчислює її розміри
  - визначає центр
  - обчислює безпечну дистанцію
  - ставить камеру
  - оновлює параметри `near/far`
  - оновлює `controls`
  - масштабує сітку та осі
  - зберігає новий стан камери

#### `updateGridScale(modelSize, modelCenter)`

Підлаштовує розмір сітки й осей під модель.

### 6.10. Режими відображення

#### `applyWireframeMode(isWireframe)`

Вмикає або вимикає wireframe-режим.

#### `applyMathStyleMode(isEnabled)`

Вмикає або вимикає математичний стиль.

У математичному стилі:

- грані напівпрозорі
- видимі ребра суцільні
- приховані ребра штрихові

#### `syncRenderModeControls()`

Синхронізує математичний режим і wireframe, щоб вони не конфліктували.

#### `enableMathStyle(mesh)`

Вмикає математичний стиль для одного mesh.

#### `disableMathStyle(mesh)`

Повертає стандартний матеріал для одного mesh.

#### `createMathFaceMaterial(sourceMaterial)`

Створює матеріал для математичного стилю.

#### `ensureMathEdgeHelpers(mesh)`

Додає допоміжні ребра до mesh:

- суцільні видимі
- штрихові приховані

### 6.11. Thumbnail-прев'ю

#### `enqueueThumbnail(asset)`

Додає модель у чергу створення прев'ю.

#### `processThumbnailQueue()`

Послідовно обробляє чергу thumbnail, щоб не навантажувати браузер одночасними рендерами.

#### `createAssetThumbnail(arrayBuffer)`

Створює PNG-прев'ю моделі у форматі `data URL`.

#### `prepareThumbnailModel(modelRoot)`

Спрощує матеріали моделі для прев'ю.

#### `frameThumbnailCamera(previewCamera, modelRoot)`

Підбирає камеру для гарного вигляду thumbnail.

### 6.12. Статистика, статус і підказка

#### `collectModelStats(modelRoot)`

Рахує:

- кількість вершин
- кількість трикутників

#### `updateStats({ status, name, size, vertices, polygons })`

Оновлює блок статистики моделі в інтерфейсі.

#### `setStatus(message)`

Оновлює короткий текстовий статус у верхній панелі.

#### `hideSceneHint()`

Ховає підказку на сцені та зберігає це рішення в `localStorage`.

#### `restoreSceneHintState()`

Відновлює стан підказки після перезавантаження сторінки.

### 6.13. Допоміжні утиліти

#### `formatFileSize(byteCount)`

Форматує розмір файлу в зручний рядок:

- `B`
- `KB`
- `MB`
- `GB`

#### `isGlbFile(fileName)`

Перевіряє, чи є файл `.glb`.

#### `createSessionFileKey(file)`

Створює унікальний ключ локального файлу для уникнення дублювання.

### 6.14. Очищення ресурсів

#### `disposeActiveModel()`

Очищає поточну модель зі сцени та звільняє пов'язані ресурси.

Що очищає:

- геометрії
- звичайні матеріали
- оригінальні матеріали
- математичні матеріали
- текстури

#### `disposeThreeObject(root)`

Звільняє ресурси довільного дерева `Three.js` об'єктів.

#### `disposeMaterial(material)`

Звільняє текстури та сам матеріал.

### 6.15. Адаптивність і layout

#### `resizeRenderer()`

Перераховує розмір renderer, canvas, камери та viewport gizmo.

#### `syncMobileViewportHeight()`

Коригує висоту canvas на мобільних екранах у режимі перегляду.

#### `syncViewerLayout(options = {})`

Декілька разів синхронізує layout після змін інтерфейсу, щоб уникати помилок рендеру на мобільних пристроях.

### 6.16. Обробка помилок і цикл рендеру

#### `handleLoadError(file, error)`

Єдина точка обробки помилки завантаження моделі.

Що робить:

- виводить помилку в консоль
- формує зрозумілий текст для користувача
- оновлює статус
- оновлює блок статистики

#### `onPointerMove(event)`

Оновлює координати курсора для `raycaster`.

Поточне призначення:

- підготовка до майбутніх інструментів вибору точок
- основа для можливих вимірювань або інтерактивного виділення

#### `animate()`

Основний нескінченний цикл рендеру.

Що робить:

- викликає сам себе через `requestAnimationFrame`
- оновлює `OrbitControls`
- синхронізує поворот gizmo
- очищає кадр
- рендерить основну сцену
- домальовує gizmo

#### `renderViewGizmo()`

Рендерить mini-gizmo поверх основної сцени в окремому viewport.

- перемикач режиму `Розгортка`
- кнопку `Розгорнути / Пауза / Згорнути`
- повзунок прогресу анімації `0% ... 100%`
- окрему навчальну сцену розгортки поверх звичайної моделі
- анімований перехід від складеної фігури до плоскої розгортки

### Що підтримується в поточній версії

- `Куб` — розгортка у вигляді хреста
- `Піраміда` — розгортка з квадратною основою та чотирма бічними трикутниками

### Що поки не підтримується

- довільні імпортовані `.glb` моделі
- автоматичне обчислення розгортки для будь-якого многогранника
- розгортка для `Куб зі зрізом`
- розгортка для `Циліндр`
- розгортка для `Сфери`

### Як реалізовано

Поточна версія не розбирає довільний меш на грані автоматично. Замість цього для підтриманих навчальних фігур створюється спеціальна допоміжна геометрія розгортки, яка:

- має окремі грані
- плавно переходить між складеним і розгорнутим станом
- рендериться прямо в основній 3D-сцені
- використовує той самий viewer, камеру та навігацію

### Нові функції, додані для `Unfold Mode`

#### `getCurrentDisplayRoot()`

Повертає об'єкт, який зараз фактично показується користувачу:

- або звичайну модель
- або групу розгортки

#### `refreshUnfoldController()`

Перебудовує допоміжний контролер розгортки після завантаження нової моделі.

#### `disposeUnfoldController()`

Видаляє поточну розгортку та звільняє її ресурси.

#### `getSupportedUnfoldType(asset)`

Визначає, чи підтримує активна модель режим розгортки, і повертає тип фігури.

#### `syncUnfoldVisibility()`

Перемикає видимість:

- звичайної моделі
- моделі розгортки

#### `setUnfoldModeEnabled(isEnabled)`

Вмикає або вимикає режим розгортки.

#### `toggleUnfoldPlayback()`

Запускає або зупиняє анімацію розгортання чи згортання.

#### `onUnfoldProgressInput(event)`

Дозволяє вручну встановлювати прогрес розгортки через повзунок.

#### `updateUnfoldAnimation(deltaSeconds)`

Оновлює стан анімації розгортки в кожному кадрі.

#### `applyUnfoldRenderStyle()`

Підлаштовує стилі граней розгортки під поточні режими відображення.

#### `updateUnfoldUiState()`

Оновлює стан елементів керування розгорткою в інтерфейсі.

#### `buildUnfoldController(unfoldType, modelRoot)`

Створює потрібний контролер розгортки залежно від типу фігури.

#### `buildCubeUnfoldController(size)`

Формує розгортку куба у вигляді хреста.

У поточній реалізації куб розгортається не простим лінійним зміщенням граней, а через шарнірні повороти навколо ребер. Це дає більш правильну навчальну анімацію "розкриття" куба.

#### `buildSquarePyramidUnfoldController(size)`

Формує розгортку квадратної піраміди.

#### `createUnfoldControllerFromFaces(faceDefinitions)`

Створює універсальний контролер розгортки з набору граней і трансформацій.

#### `createUnfoldFace(geometry)`

Створює одну грань розгортки з матеріалом і контуром.

#### `createTriangleFaceGeometry(baseWidth, height)`

Створює трикутну геометрію для бічних граней піраміди.

#### `createFaceTransform(position, xDirection, yDirection)`

Будує трансформацію грані за локальними осями.

### Які наявні функції було змінено

#### `bindEvents()`

Доповнено подіями для:

- перемикача розгортки
- кнопки запуску анімації
- повзунка прогресу

#### `loadAsset(asset, options = {})`

Тепер скидає стан попередньої розгортки під час переходу на нову модель.

#### `parseModelBuffer(arrayBuffer, asset, options = {})`

Після завантаження моделі створює або оновлює контролер розгортки.

#### `frameCurrentModel(options = {})`

Тепер вміє кадрувати не лише звичайну модель, а й площу розгортки.

#### `updateViewerActions()`

Тепер додатково оновлює елементи UI, пов'язані з розгорткою.

#### `disposeActiveModel()`

Тепер також очищає допоміжну сцену розгортки.

#### `animate()`

Тепер у кожному кадрі оновлює ще й анімацію розгортки.

### Важливе обмеження поточної реалізації

Поточна реалізація орієнтована саме на навчальні еталонні фігури. Для довільних GLB-моделей у майбутньому знадобиться окремий алгоритм:

- виділення граней
- пошук спільних ребер
- побудова графа суміжності
- обчислення коректної розгортки без самоперетинів
